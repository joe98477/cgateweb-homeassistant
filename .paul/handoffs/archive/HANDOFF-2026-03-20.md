## What's In Progress

**CNI Configuration Gap — the next feature to implement:**
- In managed mode, C-Gate runs locally but there is NO way to configure the CNI (C-Bus Network Interface) IP address
- User has a Wiser 2 unit at 192.168.0.200 on their LAN
- C-Gate needs to know the CNI IP to connect to the physical C-Bus network
- Codebase research confirmed: NO CNI/interface config exists in config.yaml, C-GateConfig.txt setup, or anywhere else
- Web research on C-Gate docs was interrupted — need to find the exact C-GateConfig.txt properties for CNI connection

**Key finding from codebase research:**
- Config options flow: `config.yaml` → `/data/options.json` → shell scripts & ConfigLoader.js
- C-GateConfig.txt is written by both `cgate-install.sh` and `cgate/run` service script
- Currently only sets: `project.default`, `cgate.port`, `cgate.event.port`
- Missing: CNI/interface IP address, interface type, serial port settings
